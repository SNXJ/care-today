## 1. 项目定位

“陪你一起过今天”是一个面向乳腺癌患者、家人和朋友的陪伴支持 Web 工具。项目重点不是提供医疗诊断或治疗建议，而是帮助患者和家属整理日常安排、复诊问题、身体状态、帮忙任务和陪伴留言。

本项目第一阶段定位为：**可部署到服务器和域名的 Vue 3 Web 应用，兼容手机和电脑访问，数据存储在服务端数据库。**

## 2. 核心目标

- 帮患者减少记事压力：记录复诊、检查、身体状态、想问医生的问题。
- 帮家属把关心变成具体行动：陪诊、做饭、接送、买药、整理资料等任务可认领。
- 帮复诊更有准备：沉淀症状记录、问题清单、用药和报告资料。
- 保持安全边界：不诊断、不推荐治疗方案、不判断用药、不替代医生建议。
- 支持多人使用：同一个患者可邀请家人/朋友加入；未来也可给其他家庭使用。

## 3. 用户角色

### 3.1 患者

主要使用者，可创建自己的陪伴空间，记录身体状态、复诊安排、问题清单、资料和留言。

### 3.2 家人/朋友

被患者邀请加入，可查看被授权内容、认领帮忙任务、留言、添加陪诊安排。

### 3.3 空间管理员

默认由患者担任，也可指定一位家属。负责邀请成员、设置权限、删除成员、管理数据。

### 3.4 平台管理员

后台运维角色，用于用户管理、异常处理、内容安全、数据安全审计。第一版可以只做最小后台或命令行运维，不必做完整管理后台。

## 4. 端与技术形态

### 4.1 Web 前端

采用 Vue 3 开发，支持响应式布局：

- 手机浏览器：主要使用场景，底部导航、卡片式信息流。
- 电脑浏览器：适合家属整理资料、复诊前查看全局信息。
- 微信内置浏览器：需要兼容，但第一版不做微信登录强绑定。

### 4.2 是否使用 UniApp

当前需求是“网页 + Vue 3 + 域名访问”，建议第一版使用 **Vite + Vue 3** 先完成 Web 上线，同时把后端 API、数据结构和权限模型设计成可复用形态，便于后续顺带接入 UniApp。

原因：

- 如果目标是普通 Web 页面，Vue 3 H5 项目更直接，开发和部署成本低。
- UniApp适合后续同时做微信小程序、App、H5 多端发布。
- 第一版先验证核心功能和隐私流程，后续如果要做微信小程序，再评估 UniApp 或独立小程序项目。

结论：**第一版主线用 Vue 3 Web；UniApp 可以同步做技术预留和页面规划，但不建议和 Web 第一版并行开发完整功能。**

### 4.2.1 UniApp 规划

UniApp 可作为二期移动端版本，用于发布：

- 微信小程序
- H5 备用版本
- App 版本，暂不作为第一目标

UniApp 适合承接这些场景：

- 家人朋友通过微信快速打开和使用。
- 患者在手机上更方便记录身体状态、查看日程、接收提醒。
- 后续需要微信登录、订阅消息、模板消息或小程序分享邀请。

第一版需要为 UniApp 预留：

- 后端 API 不绑定 Web 页面，统一使用 REST JSON。
- 登录鉴权使用 token，不依赖浏览器 cookie。
- 文件访问走带权限的接口，不暴露公开文件 URL。
- 页面数据结构保持简单，便于 Web 和 UniApp 复用。
- 权限模型在后端完成，不能只靠前端隐藏按钮。

建议 UniApp 二期页面：

- 首页：今日安排、快捷求助、身体状态快捷记录。
- 日历：复诊、检查、用药提醒。
- 身体记录：每日状态和最近趋势。
- 问医生：问题清单和医生答复记录。
- 帮忙墙：任务认领和完成。
- 留言：家人朋友陪伴留言。
- 我的：成员、权限、隐私说明。

暂不建议 UniApp 第一版包含：

- 复杂资料夹文件上传。
- 多患者空间管理。
- 管理后台。
- AI 医疗问答。

### 4.3 后端与数据库

第一版需要服务端数据库，建议架构：

- 前端：Vue 3 + Vite
- 后端：Spring Boot 3 或 Node.js/NestJS 二选一
- 数据库：MySQL 或 PostgreSQL
- 文件存储：本地服务器目录或对象存储
- 部署：Nginx + HTTPS + 后端服务

如果想沿用当前仓库能力，可以优先选择 Spring Boot 3，因为仓库已有 `shop-verification` 后端结构。

## 5. 第一版功能范围

### 5.1 登录与账号

必须支持：

- 手机号/验证码登录，或账号密码登录。
- 创建陪伴空间。
- 邀请家人/朋友加入空间。
- 成员退出空间。
- 空间管理员移除成员。

暂不支持：

- 微信授权登录。
- 第三方账号登录。
- 多机构/医院账号体系。

### 5.2 陪伴空间

一个患者对应一个或多个陪伴空间。第一版建议一个患者默认一个空间。

字段示例：

- 空间名称：例如“陪你一起过今天”
- 患者昵称
- 诊疗阶段备注：可选，不强制填写
- 成员列表
- 权限设置

### 5.2.1 信息架构：输入与汇总

各功能模块的关系按「输入 → 汇总」组织：

- **输入源**：日程、身体记录、问医生清单、朋友圈动态、资料夹、注意事项。录入统一通过全局「+」发布面板或对应页面完成。
- **汇总视图**：
  - **今日首页**：只展示「今天要做的事」——今天的日程、生效中的注意事项。
  - **时间线**：展示全部动态，按时间排序，包含过去的记录和未来的计划。

### 5.2.2 导航结构（v0.3）

常驻导航只保留 5 个 tab：**今天、时间线、朋友圈、身体、注意**。

- 日程添加、问医生、存资料、说说此刻：通过页面右下角全局「+」发布面板弹窗完成。
- 问医生清单、复诊资料夹：从「今天」页复诊倒计时卡片的按钮弹窗打开。
- 成员与权限、隐私说明：从顶栏「成员 · 邀请家人」按钮弹窗打开。
- 帮忙墙：v0.3 起功能下线，不再展示和新增；历史数据保留在数据库中，后端 API 保留。

### 5.3 今日首页

首页是手机端第一屏核心页面，**只展示今天要做的事情**。

内容：

- 今天要做的事（聚合清单，按时间排序）：
  - 今天的日程（含是否需要陪同）
  - 生效中的注意事项（置顶提醒）
- 下次复诊倒计时（按下一个未来日程真实计算），附「问题清单」「复诊资料」弹窗入口
- 身体状态快捷记录
- 及时联系医生提示与医疗边界提示

不在首页展示：朋友圈动态（去朋友圈页和时间线查看）。

### 5.3.1 时间线

时间线是全量汇总视图，展示所有模块产生的动态：

- 来源：日程、身体记录、问医生、朋友圈动态、资料、注意事项。
- 排序：日程按**计划时间**，其余按**创建时间**。
- 排列：未来计划在上，「今天」分隔条，过去记录在下；进入页面自动定位到今天。
- 交互：点击事件弹出详情弹窗（完整字段和备注全文），不跳转页面；可在弹窗内直接编辑、删除。
- 展示全部记录，不做条数截断；长文字在列表内截断、详情内完整展示。

### 5.4 日历与安排

支持记录：

- 复诊
- 检查
- 治疗
- 取报告
- 用药提醒
- 家属陪同安排

字段：

- 标题
- 日期时间
- 地点
- 备注
- 是否需要陪同
- 关联成员

第一版不做短信提醒，先做站内提醒/页面提醒。

v0.3 起没有独立日历页：添加走全局「+」面板弹窗，查看在「今天」页和时间线（未来部分），编辑/删除在时间线详情弹窗中完成。

### 5.5 身体状态记录

患者可记录每日状态：

- 疼痛
- 乏力
- 睡眠
- 心情
- 食欲
- 体温
- 备注

展示：

- 今日记录（快捷滑条 + 一句感受）
- 趋势图：近 7 天 / 近 30 天可切换，疼痛、乏力、睡眠、心情、食欲、体温六个指标逐项查看（纯 SVG 折线，不引入图表库）
- 复诊前可查看最近记录

边界：

- 不根据记录给出诊断。
- 不自动判断治疗风险。
- 仅提示“如明显不适或症状加重，请及时联系医生/医院”。

### 5.6 问医生清单

支持患者和家属共同维护复诊问题。

默认问题模板：

- 我的分期、分型和治疗目标是什么？
- 当前治疗常见副作用有哪些？
- 哪些情况需要马上联系医院？
- 饮食、运动、工作、睡眠有什么限制？
- 下次检查需要提前准备什么？

功能：

- 添加问题（「+」面板或清单弹窗内添加）
- 勾选已问（同步到后端）
- 记录医生答复
- 标记重要

入口：「今天」页复诊卡片的「问题清单」按钮弹窗。

### 5.7 朋友帮忙墙（v0.3 已下线）

v0.3 起帮忙墙功能下线：前端不再展示和新增帮忙任务。历史数据保留在 `help_tasks` 表中，后端 API 保留，后续如需要可恢复。

### 5.8 朋友圈（原温柔留言）

患者的想法和状态记录，类似朋友圈：

- **仅患者本人（空间管理员）可以发布**，后端校验权限；家人朋友查看。
- 独立「朋友圈」tab：顶部发布框（仅患者可见），下方按时间倒序的动态流。
- 动态同时汇总进时间线；患者可编辑、删除自己的动态。
- 第一版仅文字，不支持图片/语音；不做点赞/评论，避免社交压力。

### 5.9 资料夹

用于整理复诊资料。

第一版支持：

- 报告名称记录
- 用药记录
- 医嘱备注
- 文件上传可作为二期功能，第一版可先只做文本资料

如果第一版就支持上传文件，必须增加：

- 文件大小限制
- 文件类型限制
- 访问权限校验
- 文件加密或私有存储
- 删除能力

### 5.10 注意事项

记录医嘱和生活禁忌类提醒，例如「化疗期间避免生食」「术后两周不要提重物」。

字段：

- 内容（必填）
- 补充说明（可选）
- 是否重要
- 生效开始/结束日期（可选，都不填表示长期有效）
- 状态：生效中、已归档

功能：

- 添加、编辑、删除（软删除）
- 归档 / 恢复
- 生效中的注意事项置顶展示在今日首页
- 添加动态展示在时间线

边界：注意事项由患者和家属自行记录医生的叮嘱，不由系统生成或判断。

### 5.11 隐私与免责声明

页面必须显示：

> 本工具仅用于生活陪伴、就诊整理和家庭协作，不提供医疗诊断、治疗建议或用药判断。涉及治疗方案、用药调整和症状处理，请以主治医生或医院意见为准。

注册/创建空间时必须提示：

- 将收集哪些信息。
- 为什么需要这些信息。
- 谁可以看到这些信息。
- 如何删除或退出。

## 6. 暂不支持范围

第一版明确不做：

- 不做在线问诊。
- 不做 AI 医疗诊断。
- 不推荐治疗方案。
- 不判断检查报告。
- 不判断用药是否正确。
- 不做病情预测。
- 不支持导出资料包。
- 不做保险、药品、医院商业推荐。

## 7. 权限设计

### 7.1 权限等级

- 患者/管理员：完整管理权限。
- 家属：可查看多数信息、创建任务、认领任务、留言。
- 朋友：可查看被授权内容、认领任务、留言。
- 只读成员：只能查看，不可编辑。

### 7.2 权限原则

- 默认最小可见。
- 新成员加入后，需要管理员授权。
- 敏感资料默认仅患者和管理员可见。
- 成员退出后，不再能访问空间数据。

## 9. 安全与合规要求

本项目会处理医疗健康相关内容，属于敏感个人信息范畴。根据《中华人民共和国个人信息保护法》，医疗健康信息属于敏感个人信息；处理敏感个人信息应有特定目的、充分必要性并采取严格保护措施。参考来源：

- [中华人民共和国个人信息保护法 - 工信部](https://www.miit.gov.cn/jgsj/zfs/fl/art/2022/art_515a4b20c12f430eab54bb4f56d89f56.html)
- [网络数据安全管理条例 - 中国政府网](https://app.www.gov.cn/govdata/gov/202409/30/520076/article.html)
- [个人信息保护政策法规问答 - 中央网信办](https://www.cac.gov.cn/2026-01/09/c_1769688003183197.htm)

第一版必须做到：

- 全站 HTTPS。
- 密码加密存储，不保存明文密码。
- 后端接口必须鉴权。
- 空间数据必须按成员权限隔离。
- 敏感操作记录审计日志。
- 用户可删除自己的账号或退出空间。
- 空间管理员可移除成员。
- 数据库定期备份。
- 不在前端暴露服务端密钥。
- 不把病情数据写入公开日志。

建议做到：

- 重要字段加密存储。
- 文件私有存储，不允许公开 URL 直接访问。
- 邀请链接设置有效期。
- 防止暴力登录。
- 管理员后台启用强密码。

## 12. 部署与 Docker 方案

### 12.1 是否使用 Docker

建议使用 Docker Compose 管理后端和数据库，前端 Vue 3 打包后可直接由 Nginx 托管。

推荐部署结构：

- Nginx：负责 HTTPS、域名访问、前端静态文件、`/api` 反向代理。
- Backend：后端 API 服务，建议容器化。
- Database：MySQL 或 PostgreSQL，建议容器化并挂载数据卷。
- Redis：第一版暂不需要，后续短信验证码、提醒任务、频繁缓存时再引入。

### 12.2 推荐目录

```text
support-page/
  frontend/              Vue 3 Web
  uniapp/                UniApp 二期移动端，先预留
  backend/               后端 API 服务
  deploy/
    docker-compose.yml
    nginx.conf
    backup.sh
  docs/
    需求文档.md
```

### 12.3 部署要求

- 域名必须配置 HTTPS。
- 数据库必须挂载持久化目录。
- 数据库必须定期备份。
- 后端配置通过环境变量注入，不把数据库密码、JWT 密钥写死在代码里。
- 生产环境关闭调试日志，避免敏感信息进入日志。
- 前端请求统一走 `/api`，由 Nginx 反向代理到后端。

### 12.4 第一版 Docker Compose 组件

第一版建议包含：

- `support-backend`
- `mysql` 或 `postgres`
- `nginx`

暂不包含：

- Redis
- 消息队列
- 对象存储
- 独立管理后台容器

## 14. 验收标准

第一版上线前应满足：

- 用户可以注册/登录。
- 用户可以创建陪伴空间。
- 可以邀请家人/朋友加入。
- 不同空间数据互不可见。
- 可以创建、查看、编辑、删除日程。
- 可以记录身体状态。
- 可以维护问医生清单。
- 患者可以在朋友圈发布动态，家人只能查看（后端校验）。
- 可以记录、归档注意事项。
- 首页只展示今天的日程和生效中的注意事项。
- 时间线展示全部动态，未来在上、今天分隔、过去在下，点击弹出详情不跳转，可在详情中编辑删除。
- 身体页可切换 7/30 天查看各项指标趋势。
- 手机端页面可正常使用。
- HTTPS 正常。
- 关键接口需要登录。
- 无权限用户无法访问空间数据。
- 页面明确展示医疗边界和隐私说明。
