# 设计文档：发布入口统一 + 症状记录 + 体重趋势

日期：2026-06-11
状态：已确认（用户口述需求，方向已批准）
关联需求文档：`患者陪伴需求文档.md` v0.4

## 1. 需求来源（用户原话归纳）

1. 把所有发布功能都放到「+」面板内。
2. 主 tab 的今天、时间线、分享（朋友圈改名为分享）只展示信息。
3. 身体 tab 只展示各指标数据变化，增加体重统计变化；该 tab 内「+」只添加身体相关。
4. 增加记录"今日什么时间大便、什么时间发烧"（快捷标签：大便、腹泻、发烧、乏力、疼痛、手脚发麻，支持自定义）。
5. 各板块提示语差异化，不要都一样。

## 2. 发布入口

全局「+」面板七个动作，全部弹窗完成，不跳页面：

说说此刻（仅患者）· 加日程 · 记身体状态 · 记症状 · 记注意事项 · 问医生的问题 · 存一条资料

- 「记身体状态」弹窗：五项评分滑条 + 体温 + **体重（可不填）** + 一句感受。
- 「记症状」弹窗：快捷标签 chips + 「自定义…」输入 + 发生时间（默认当前，datetime-local）+ 补充说明。
- 「记注意事项」改为表单弹窗（内容/说明/起止日期/重要）。
- 在「身体」tab 打开「+」时只显示记身体状态、记症状两项。

各 tab 上的内联表单全部移除：分享发布框、注意事项页表单、身体页滑条卡片。

## 3. 数据（Flyway V5，纯增量）

- `body_records` 增加 `weight DECIMAL(5,1) NULL`。
- 新表 `symptom_events`：id、space_id、tag、happened_at、note、created_by、created_at/updated_at/deleted_at（软删除）。
- API：`/api/spaces/{id}/symptoms` GET/POST/PATCH/DELETE，空间成员权限 + 审计日志。

## 4. 展示

- **身体 tab**：趋势图（指标增加体重，量程按数据动态计算 min-1 / max+1）+ 症状记录（今天 / 最近 7 天两段列表，行内编辑删除）。
- **时间线**：症状作为新类型（按发生时间排序），详情弹窗可编辑（标签/说明）和删除；身体记录详情新增体重字段，也支持编辑（感受）/删除。
- **今天页**：移除身体快捷记录卡片，纯展示（日程 + 生效注意事项 + 倒计时）。
- **分享 tab**：纯动态流，标签提示发布入口在「+」。

## 5. 提示语差异化（节选）

| 位置 | 文案 |
| --- | --- |
| 今天空 | 今天没有安排，休息也很重要。日程到了当天会自动出现在这里。 |
| 时间线空 | 时间线还空着。点右下角「+」记第一笔，所有记录都会按时间在这里排好。 |
| 分享空（患者） | 还没有分享过。点右下角「+」说说此刻的想法和状态，家人打开就能看到。 |
| 分享空（家属） | TA 还没有发过分享，过两天再来看看。 |
| 趋势不足 | 至少要有两天的记录才能连成线。在这个页面点「+」记身体状态，曲线会自己长出来。 |
| 症状空 | 今天还没有症状记录。在这个页面点「+」选「记症状」，时间会自动带上。 |
| 注意空 | 还没有注意事项。点右下角「+」选「记注意事项」，比如「化疗期间避免生食」。 |

## 6. 验收

- `mvn test`、`npm run build` 通过。
- 症状增删改查 e2e 正常；体重写入并出现在趋势图。
- 主 tab 无任何输入表单；身体 tab 的「+」只显示身体相关两项。
