# 设计文档：导航精简 + 朋友圈 + 身体趋势图

日期：2026-06-11
状态：已确认（用户已批准方向，朋友圈独立 tab 为用户补充修正）
关联需求文档：`患者陪伴需求文档.md` v0.3

## 1. 背景与目标

v0.2 后导航有 10 个 tab，移动端拥挤；留言/帮忙等模块使用率与定位不匹配。本次改版：

- 常驻导航精简为 5 个 tab：**今天、时间线、朋友圈、身体、注意**；
- 录入统一走**全局「+」发布面板**（FAB 弹窗）；
- 「温柔留言」升级为**朋友圈**独立页面：患者发想法和状态，家人查看；
- **帮忙墙功能下线**（历史数据和后端 API 保留）；
- 身体页新增 **7/30 天趋势图**。

时间线保持 v0.2 形态不变（全量汇总、未来在上、今天分隔、点击弹详情），朋友圈动态作为一种记录类型汇入时间线。

## 2. 导航与入口

| 功能 | v0.2 入口 | v0.3 入口 |
| --- | --- | --- |
| 日程添加 | 日历页表单 | 全局「+」面板弹窗（标题/时间/地点/备注/陪同） |
| 日程查看/编辑/删除 | 日历页 | 今天页、时间线（详情弹窗内编辑/删除） |
| 问医生 | 问医生页 | 「+」面板快速添加；今天页复诊卡片「问题清单」弹窗管理 |
| 留言 | 留言页 | 朋友圈 tab（仅患者发布） |
| 帮忙墙 | 帮忙墙页 | 下线 |
| 资料夹 | 资料夹页 | 「+」面板存资料；今天页复诊卡片「复诊资料」弹窗管理 |
| 成员 | 成员页 | 顶栏「成员 · 邀请家人」弹窗（含隐私说明、退出空间、删除账号） |

## 3. 朋友圈

- 独立 tab，位于时间线之后。
- 顶部发布框：**仅患者（PATIENT_ADMIN）可见**；家属看到"只有患者本人可以发布"提示。
- 动态流按时间倒序；患者可编辑/删除自己的动态。
- 后端 `createMessage` 增加角色校验（403：只有患者本人可以发布动态）；`getSpace` 返回 `currentRole` 供前端判断。
- 不做点赞/评论，避免社交压力。

## 4. 详情弹窗增强

时间线详情弹窗对可编辑类型（日程/动态/资料/注意/问医生）显示「编辑」「删除」按钮，复用各自的表单弹窗和确认流程；身体记录在身体页管理。

## 5. 身体趋势图

- 数据：按 `recordDate` 每天取最新一条，最近 7 或 30 天。
- 指标：疼痛/乏力/睡眠/心情/食欲（0–10 分）、体温（34–42℃），单指标切换查看。
- 实现：纯 SVG（polyline + circle），无第三方图表库；少于 2 个数据点时显示引导文案。

## 6. 其他实现要点

- 通用表单弹窗（openFormDialog）扩展字段类型：`datetime-local`、`date`、`checkbox`、placeholder。
- FAB：右下角固定，移动端上移避开底部导航。
- 「勾选已问」修复：原问医生页 checkbox 只改本地状态不落库，现同步调用 `updateDoctorQuestion({ asked })`。
- 帮忙相关前端代码（页面、状态、API 调用、map 函数）全部移除；后端及数据库不动。
