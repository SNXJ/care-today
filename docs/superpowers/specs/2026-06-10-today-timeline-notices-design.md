# 设计文档：今天页改版 + 时间线重做 + 注意事项模块

日期：2026-06-10
状态：已确认（用户已批准）
关联需求文档：`患者陪伴需求文档.md` v0.2

## 1. 背景与目标

第一版上线后，信息分散在各模块，首页和时间线没有形成「输入 → 汇总」的清晰关系。本次改版确立：

- **日历、身体、问医生、帮忙墙、留言、资料夹、注意事项都是输入源**；
- **首页「今天」只展示今天要做的事**（今天的聚合视图）；
- **时间线展示全部按时间排序的动态**（过去 + 未来的全量视图）；
- 新增**注意事项**（医嘱/生活禁忌类）记录。

数据完全保留：数据库只做增量迁移（Flyway V2），不改动既有表和数据。

## 2. 首页「今天」

主卡片改为「今天要做的事」聚合清单，按时间排序，包含：

- 今天的日程（`events` 中计划时间是今天的）；
- 今天的帮忙任务（`help_tasks` 中约定时间是今天的，显示状态与认领人）；
- 生效中的注意事项（⚠ 条形提示，amber 风格，置于清单顶部，点击弹详情）。

保留：快捷发起、「今天身体怎么样」快捷记录、侧栏「及时联系医生」。
移除：留言卡片、不限日期的待认领任务卡片（从导航和时间线查看）。

修复：「距离下次复诊」从写死的 9 改为按下一个未来日程真实计算天数；没有未来日程时显示引导文案。

空状态：今天没有安排时显示温柔文案（如「今天没有安排，好好休息」）。

## 3. 时间线

- **排列**：未来计划在上（时间越远越靠上），「今天 · X月X日」分隔条，过去记录在下（时间越近越靠上）。进入页面自动滚动定位到「今天」分隔条附近。
- **数据**：取消 30 条上限，展示全部。日程和帮忙任务按**计划时间**（scheduledAt）排序，身体/问医生/留言/资料/注意事项按**创建时间**排序。
- **点击行为**：弹出详情弹窗（复用现有弹窗视觉），展示完整标题、带年份的日期时间、地点/分数/状态/认领人等字段、备注全文。底部「知道了」关闭，次要按钮「去对应页面处理」可跳转（默认不跳转）。
- **UI 修复**：
  - 长标题/长备注溢出：flex/grid 子项补 `min-width: 0`，长词折行（`overflow-wrap: anywhere`），列表内 detail 做行数截断，全文在详情弹窗查看；
  - 日期标签跨年时补年份，并显示时刻。

## 4. 注意事项（新模块）

定位：医嘱/生活禁忌类提醒，例如「化疗期间避免生食」「术后两周不要提重物」。

### 4.1 数据库（Flyway `V2__care_notices.sql`，纯增量）

`care_notices` 表：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | CHAR(36) | 主键 UUID |
| space_id | CHAR(36) | 所属陪伴空间，外键 |
| content | VARCHAR(200) | 注意事项内容，必填 |
| detail | TEXT | 补充说明，可选 |
| important | BOOLEAN | 是否重要，默认 false |
| starts_on | DATE | 生效开始日期，可选 |
| ends_on | DATE | 生效结束日期，可选；两者都空 = 长期有效 |
| status | ENUM('active','archived') | 生效中 / 已归档 |
| created_by | CHAR(36) | 创建人，外键 users |
| created_at / updated_at / deleted_at | DATETIME(3) | 与其他表一致，软删除 |

### 4.2 API

`/api/spaces/{spaceId}/notices`，沿用 CareController/Service/Repository 模式与空间成员权限校验：

- `GET` 列表（按创建时间倒序）
- `POST` 创建（content 必填）
- `PATCH /{noticeId}` 更新（含归档/恢复：status 字段）
- `DELETE /{noticeId}` 软删除

全部操作写审计日志（`notice.create/update/delete`）。

### 4.3 前端

- 新导航项「注意」（复用 status-warning 图标），位于「问医生」之后；
- 页面：生效中列表 + 已归档列表 + 添加表单（内容、补充说明、重要标记、生效起止日期可选）+ 编辑/归档/恢复/删除；
- 展示位置（两处互补）：
  - 今天页：置顶 ⚠ 展示「当前生效中」的注意事项（status=active 且当天在生效期内）；
  - 时间线：展示「添加了一条注意事项」的动态（按创建时间）。

## 5. 不做的事

- 不改登录、空间、成员、邀请流程；
- 不做提醒推送；
- 不改其他模块的输入表单；
- 不动既有数据库表结构和数据。

## 6. 验收

- 后端 `mvn test` 通过（新增 notices 控制器测试）；
- 前端 `npm run build` 通过；
- 首页只显示今天的日程/任务/生效中注意事项，倒计时为真实天数；
- 时间线全量展示，未来在上、今天分隔、过去在下，点击弹详情不跳转；
- 长文字不再溢出卡片；
- 注意事项可增删改、归档，数据写入后端。
