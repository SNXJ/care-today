# CareToday 设计风格规范

版本：v0.1  
适用项目：CareToday（陪你一起过今天）  
适用范围：Web 前端、后续 Vue 3 页面、移动端 H5/UniApp 页面规划

## 1. 设计方向

CareToday 是面向患者、家人和朋友的陪伴支持工具。界面需要让人感觉稳定、清楚、被照顾，但不能显得像医疗诊断系统，也不能像营销落地页。

整体风格关键词：

- 温和
- 克制
- 清楚
- 可信
- 低压力
- 家庭协作

视觉上采用“温柔纸面 + 轻量信息面板”的方向。页面应该像一本整理好的陪伴手册，而不是医院后台系统，也不是情绪化的公益海报。

## 2. 设计原则

### 2.1 信息先于装饰

页面核心是帮助用户记录、查看和协作。所有视觉元素都应该服务于：

- 今天要做什么
- 下次复诊是什么时候
- 身体状态有没有记录
- 哪些事情需要家人朋友帮忙
- 哪些问题要问医生
- 谁可以看到这些信息

不使用大面积纯装饰图形，不使用复杂背景插画压过内容。

### 2.2 温柔但不含糊

文案和视觉都要避免“过度安慰”。界面可以柔和，但信息层级必须明确。

推荐表达：

- 今天先照顾好这一件事
- 不用一个人记住所有事情
- 把担心的问题先放在这里
- 这只是整理记录，不替代医生判断

避免表达：

- 一定会好
- 你要坚强
- 系统判断风险
- 推荐治疗方案

### 2.3 医疗边界常驻

涉及身体记录、问医生、资料夹、紧急提醒等模块时，必须保留边界提示：

> 本工具仅用于生活陪伴、就诊整理和家庭协作，不提供医疗诊断、治疗建议或用药判断。

该提示可以是卡片、侧边说明、底部说明或创建空间时的确认项，但不能完全隐藏在隐私页里。

## 3. 色彩系统

当前原型色彩可作为第一版设计 token。

### 3.1 基础色

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--paper` | `#fbf8f4` | 页面背景，柔和纸面底色 |
| `--panel` | `#fffdf9` | 主卡片背景 |
| `--panel-strong` | `#fff7ee` | 选中态、轻强调背景 |
| `--ink` | `#2e2a27` | 主文字 |
| `--muted` | `#746b63` | 次级文字 |
| `--soft` | `#9a8d82` | 辅助文字、时间、弱信息 |
| `--line` | `#eadfd3` | 分割线、边框 |

### 3.2 功能色

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--rose` | `#c76872` | 主要行动、当前状态、温柔强调 |
| `--rose-soft` | `#f7d9dc` | 主色浅背景 |
| `--sage` | `#78927c` | 安全、已保存、隐私、稳定状态 |
| `--sage-soft` | `#dfeadc` | 绿色浅背景 |
| `--amber` | `#d79a4b` | 提醒、日程、轻警示 |
| `--blue` | `#6f8da6` | 资料、文档、信息型内容 |

### 3.3 颜色使用规则

- 主色使用 `rose`，但不要让页面大面积变粉。
- `sage` 用于隐私、安全、已完成、稳定状态。
- `amber` 只用于提醒和日程强调，不用于危险警告。
- 医疗紧急提示使用浅暖底色和明确文字，不用强烈红色制造恐慌。
- 页面整体不能变成单一粉色、单一米色或单一绿色。
- 禁止使用高饱和霓虹色、紫蓝渐变、重医疗蓝白风格。

## 4. 字体与排版

### 4.1 字体栈

标题字体：

```css
font-family: Georgia, "Times New Roman", "Songti SC", serif;
```

正文字体：

```css
font-family: "Avenir Next", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
```

### 4.2 字号建议

| 层级 | 桌面端 | 移动端 | 用途 |
| --- | --- | --- | --- |
| 页面主标题 | 44-54px | 32-38px | 首页名称、核心页面标题 |
| 页面副标题 | 15-16px | 14-15px | 说明文字 |
| 卡片标题 | 20-22px | 18-20px | 模块标题 |
| 正文 | 14-15px | 14px | 卡片内容 |
| 辅助文字 | 12-13px | 12px | 时间、说明、状态 |

### 4.3 排版规则

- `letter-spacing` 保持 `0`，不使用负字距。
- 正文行高建议 `1.6-1.8`。
- 中文长文必须自动换行，不允许横向溢出。
- 页面标题可以使用 serif，功能控件和表单使用 sans。
- 卡片内标题不要使用过大的 hero 字号。

## 5. 布局规范

### 5.1 桌面端

桌面端采用左侧导航 + 主内容网格。

建议结构：

```text
sidebar 244px
main    minmax(0, 1fr)
```

主内容区：

- 最大内容不强行居中成窄栏，桌面端要适合家属整理资料。
- 首页可采用 `1.35fr + 0.65fr` 两栏布局。
- 重要操作在左侧主列，提醒、帮忙墙、留言等可放右侧辅助列。

### 5.2 移动端

移动端是第一主要使用场景。

规则：

- 使用底部导航。
- 主内容单列。
- 页面底部为导航预留安全距离。
- 主要按钮高度不低于 `44px`。
- 表单单列显示。
- 卡片之间间距保持 `12-18px`。
- 不允许横向滚动。

### 5.3 间距

| Token | 值 | 用途 |
| --- | --- | --- |
| `--space-1` | `6px` | 细小间隔 |
| `--space-2` | `10px` | 图标文字间距 |
| `--space-3` | `12px` | 卡片内小块间距 |
| `--space-4` | `18px` | 卡片内边距、模块间距 |
| `--space-5` | `28px` | 页面主边距 |

## 6. 组件规范

### 6.1 卡片

卡片是主要信息容器。

```css
background: rgba(255, 253, 249, 0.94);
border: 1px solid #eadfd3;
border-radius: 8px;
box-shadow: 0 14px 44px rgba(82, 63, 47, 0.07);
```

规则：

- 卡片圆角统一 `8px`。
- 不做大圆角气泡感。
- 不把卡片嵌套在另一张卡片里。
- 卡片标题区包含图标切图、标题和可选状态标签。
- 卡片内容区要能快速扫描。

### 6.2 按钮

按钮分为三类：

主操作：

- 用于创建、保存、认领、邀请。
- 高度不低于 `44px`。
- 背景可使用 `rose` 或浅 `rose-soft`。

次操作：

- 用于查看、整理、编辑。
- 使用浅背景 + 边框。

危险操作：

- 用于删除、移除成员、退出空间。
- 需要二次确认。
- 不使用刺眼大红，优先使用清楚文字和边框警示。

### 6.3 标签

标签用于状态和分类。

示例：

- 需要陪同
- 待认领
- 已认领
- 仅管理员可见
- 复诊前

规则：

- 高度建议 `26px`。
- 字号 `12px`。
- 背景白色或浅色。
- 不用于承载长句。

### 6.4 表单

表单用于记录身体状态、留言、任务、问题。

规则：

- 输入框高度不低于 `38px`，移动端重要输入不低于 `44px`。
- 占位文案要具体，但不能制造压力。
- 保存后给出轻量 toast。
- 敏感字段需要明确谁可见。

### 6.5 Toast

Toast 用于短反馈。

示例：

- 已保存
- 已加入帮忙墙
- 先写一点今天的感受

规则：

- 不用于医疗判断。
- 停留时间建议 `1600-2200ms`。
- 位置桌面端右下，移动端底部导航上方。

## 7. 图标与切图规范

后续图标统一使用切图资产，不在页面内手绘 SVG。

### 7.1 资产格式

推荐格式：

- SVG 文件作为独立图片资产
- PNG 作为兼容备用
- WebP 用于插画或大图

不允许：

- 在 HTML/Vue 模板内直接写手绘 `<svg>`
- 每个页面临时绘制不同风格图标
- 使用 emoji 代替功能图标

### 7.2 推荐目录

```text
src/assets/icons/
  nav-today.svg
  nav-calendar.svg
  nav-doctor.svg
  nav-body.svg
  nav-help.svg
  nav-message.svg
  nav-folder.svg
  action-hospital.svg
  action-task.svg
  action-chat.svg
  status-privacy.svg
  status-warning.svg
```

### 7.3 使用方式

Vue 页面中使用图片资源：

```vue
<img class="icon" src="@/assets/icons/nav-calendar.svg" alt="" aria-hidden="true" />
```

CSS：

```css
.icon {
  width: 18px;
  height: 18px;
  object-fit: contain;
  flex: 0 0 auto;
}
```

规则：

- 装饰性图标使用 `alt="" aria-hidden="true"`。
- 表达状态的图标必须配合文字，不单独依赖图标传达含义。
- 导航图标尺寸 `20px`，卡片标题图标 `18px`，快捷操作图标 `22-24px`。
- 同一批图标需要统一线宽、端点、圆角和视觉重量。

## 8. 页面模式

### 8.1 今日首页

首页是移动端第一屏核心。

必须出现：

- 今日安排
- 下次复诊
- 今天最需要什么
- 待认领帮忙任务
- 身体状态快捷记录
- 医疗边界提示

首页不要做成营销 hero。第一屏要直接可用。

### 8.2 日历页

用于复诊、检查、治疗、取报告、用药提醒和陪同安排。

设计重点：

- 时间清楚
- 是否需要陪同清楚
- 地点和资料提示清楚
- 支持列表视图优先，日历月视图可作为辅助

### 8.3 身体记录页

用于记录疼痛、乏力、睡眠、心情、食欲、体温和备注。

设计重点：

- 打分控件要容易点选。
- 最近 7 天趋势要用于回顾，不做诊断。
- 异常提示只能引导联系医生，不能判断风险等级。

### 8.4 问医生清单页

用于复诊前整理问题和医生答复。

设计重点：

- 问题、是否已问、医生答复分区明确。
- 重要问题可以标记。
- 默认模板语气中性、具体。

### 8.5 帮忙墙页

用于家人朋友认领任务。

设计重点：

- 任务类型清楚。
- 时间和说明清楚。
- 认领人清楚。
- 状态清楚：待认领、已认领、已完成、已取消。

### 8.6 留言页

用于成员写陪伴留言。

设计重点：

- 文案温柔但不施压。
- 不鼓励承诺治疗结果。
- 支持删除自己的留言。

### 8.7 成员与权限页

用于邀请成员、调整角色、移除成员。

设计重点：

- 谁能看到什么必须明确。
- 敏感资料默认仅患者和管理员可见。
- 移除成员和退出空间需要二次确认。

## 9. 可访问性与交互

- 所有按钮必须可通过键盘聚焦。
- 表单控件必须有可访问名称。
- 图标不能作为唯一信息来源。
- 颜色对比度需满足正常阅读。
- 移动端点击区域不低于 `44px`。
- 禁止自动播放声音或强提醒动画。
- 动效保持轻量，避免造成焦虑。

## 10. 禁止事项

视觉禁止：

- 大面积医疗蓝白后台风格
- 过度粉色公益海报风格
- 紫蓝渐变科技风
- 卡片套卡片
- 大圆角糖果风
- 纯装饰图案堆叠
- 页面内手绘 SVG 图标

内容禁止：

- 暗示诊断结果
- 推荐治疗方案
- 判断用药正确性
- 预测病情
- 使用“你一定要坚强”“一定会好”等施压式表达

交互禁止：

- 无确认删除敏感数据
- 只靠前端隐藏按钮实现权限
- 上传文件后公开 URL 直接访问
- 在公开日志中记录病情或身份信息

## 12. 后续执行要求

后续新增页面时，应先确认：

- 是否符合本规范的色彩、圆角、字体和间距。
- 图标是否来自统一切图资产。
- 页面是否保留医疗边界。
- 移动端是否可单列使用。
- 是否没有横向溢出。
- 权限和隐私文案是否清楚。

如果需求与本规范冲突，优先保证医疗边界、隐私安全和移动端可用性。
